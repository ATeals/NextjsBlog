# Set

---

set 객체는 중복되지 않는 유일한 값들의 집합이다.

| 구분                                 | 배열 | Set |
| ------------------------------------ | ---- | --- |
| 동일한 값을 중복하여 포함할 수 있다. | ⭕️  | ❌  |
| 요소 순서에 의미가 있다.             | ⭕️  | ❌  |
| 인덱스로 요소에 접근할 수 있다.      | ⭕️  | ❌  |

Set은 수학적 집합을 구현하기 위한 자료구조다.

## Set 의 생성

---

set 객체는 new 연산자를 이용해 생성한다.

set 생성자 함수는 이터러블을 인수로 전달받아 객체를 생성한다. 이때 중복된 이터러블의 값은 저장되지 않는다.

```jsx
const set1 = new Set([1, 2, 2, 3]);
console.log(set1); // [1,2,3]

const set2 = new Set("hello");
console.log(set2); // ["h","e","l","o"]
```

set을 이용하여 배열의 중복된 요소를 제거할 수 있다.

```jsx
const uniq = (array) => array.filter((v, i, self) => self.indexOf(v) === i);

const uniq = (array) => [...new Set(array)];
```

## 요소 개수 확인

---

set의 개수를 확인 할때는 size 프로퍼티를 사용한다.

size 프로퍼티는 setter함수 없이 getter함수만 존재하는 접근자 프로퍼티이기 때문에 size에 숫자를 할당하여 set 객체의 요소 개수를 변경할 수는 없다.

## 요소 추가

---

set 객체에 요소를 추가할 때는 add 메서드를 사용한다.

```jsx
const set = new Set();

set.add(1);
```

add 메서드는 새로운 요소가 추가된 Set() 객체를 반환하기 때문에 연속으로 호출 할 수 있다.

```jsx
const set = new Set();

set.add(1).add(2);
```

물론 중복된 요소는 혀용하지 않기 때문에 무시한다.

NaN은 자기자신과 다른 값이다.

하지만 set에서는 서로 같다고 평가하여 중복 추가를 허용하지 않는다.

## 요소 존재 여부 확인

---

set 객체에 특정 요소에 존재여부는 has 메서드를 사용한다. has는 불리언 값을 반환한다.

```jsx
const set = new Set([1, 2, 3]);
set.has(2); // true
```

## 요소 삭제

---

set 객체에 요소를 삭제하려면 delete 메서드를 사용한다. delete는 삭제 여부를 불리언 값으로 반환한다.

set은 인덱스가 없기 때문에 삭제하려는 값을 인수로 전달해야 한다.

```jsx
const set = new Set([1, 2, 3]);
set.delete(2); // true Set(2){1,3}
```

존재하지 않는 요소를 삭제하려 하면 에러 없이 무시 된다.

## 요소 전부 삭제

---

set 객체의 모든 요소를 삭제하려면 clear메서드를 사용한다. clear는 항상 undefined를 반환한다.

```jsx
const set = new Set([1, 2, 3]);
set.clear();
```

# Map

---

map 객체는 키와 값의 쌍으로 이루어진 컬렉션이다. map은 객체와 유사하지만 다음과 같은 차이가 있다.

| 구분                   | 객체                    | Map 객체              |
| ---------------------- | ----------------------- | --------------------- |
| 키로 사용할 수 있는 값 | 문자열 또는 심벌 값     | 객체를 포함한 모든 값 |
| 이터러블               | ❌                      | ⭕️                   |
| 요소개수확인           | Object.keys(obj).length | map.size              |

## Map 객체 생성

---

생성자 함수로 생성하고 인수를 전달하지 않으면 빈 객체가 생성된다.

Map 생성자 함수의 인수로 전달한 이터러블에 중복된 키를 갖는 요소가 존재하면 값이 덮어 써진다. 따라서 Map객체에는 중복된 키를 갖는 요소가 존재할 수 없다.

```jsx
const map = new Map(["key", "value1"], ["key", "value2"]);

console.log(map); // Map(1) => {"key"=> "value2"}
```

## 요소 개수 확인

---

Map 객체의 요소 개수를 확인할 때에는 size프로퍼티를 사용한다.

map 또한 set 과 마찬가지로 size가 접근자 프로퍼티이므로 size에 숫자를 할당하여 객체 요소의 개수를 변경할 수 없다.

## 요소 추가

---

Map 객체에 요소를 추가할 때에는 set메서드를 사용한다.

```jsx
const map = new Map();

map.set("key", "value");
```

set메서드 또한 새로운 요소가 추가된 map 객체를 반환하므로 연속으로 호출할 수 있다.

```jsx
const map = new Map();

map.set("key", "value").set("key2", "value2");
```

Map 객체는 중복된 키 값을 허용하지 않기 때문에 중복된 키값을 추가하면 값이 덮어써진다.

Map도 Set과 마찬가지로 NaN값을 중복된 값이라고 판단한다.

Map의 가장 큰 특징은 키값으로 어떤 값이든 사용할 수 있다는 것이다.

## 요소 취득

---

map에서 요소를 취득하려면 get 메서드를 사용한다.

map 객체에서 인수로 전달한 키를 갖는 요소가 없으면 undefined를 반환한다.

## 요소 존재 여부

---

map객체에 특정요소가 존재하는 지 확인하려면 has 메서드를 사용한다.

## 요소 삭제

---

map 객체에 특정요소를 삭제하려면 delete메서드를 사용한다.

## 요소 일괄 삭제

---

map 객체의 모든 요소를 삭제하려면 clear메서드를 사용한다.
